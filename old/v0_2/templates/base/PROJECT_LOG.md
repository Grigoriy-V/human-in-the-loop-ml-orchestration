# Project Log

This file records important completed milestones, investigations, and project
decisions. It is not a task ledger or a detailed experiment report.

## Maintenance

- Add an entry after an important result, decision, or closed roadmap step.
- Keep one `# Project Log` heading.
- Use one dated, searchable title.
- Record the outcome and decision; include the important limitation.
- Link to detailed evidence instead of copying commands, metrics, or raw
  output.
- Do not add an entry for every implementation action or test rerun.

## Entry format

```markdown
## YYYY-MM-DD: Short milestone

### Outcome

What materially happened and where the evidence is.

### Decision

What was accepted, stopped, deferred, or selected, including the important
limitation or boundary.
```

Add `### Goal` only when the reason is not already clear from the title and
context.
